Description

In this assignment you will be asked to implement two Reliable Data Transmission Protocols. This project will be completed using C/C++ over a network emulator already provided. No knowledge of Unix system calls will be needed to complete this project.

Details

  1. Detailed project description can be found here
  2. In this laboratory programming assignment, you will be writing the sending and receiving transport-level code for implementing a simple reliable data transfer protocol
  3. Your code will have to execute in a simulated hardware/software environment. Programming interface provided to your routines, i.e., the code that would call your entities from above and from below is very close to what is done in an actual UNIX environment. Stopping/starting of timers are also simulated, and timer interrupts will cause your timer handling routine to be activated
  4. There are two phases to this lab. Phase 1: Alternating-Bit-Protocol version and Phase 2: Go-Back-N version
  5. This assignment is worth - 100 points
  6. Make sure you read the "helpful hints" and Q/A section in project description

Submission

As a part of the submission you need to provide the following:
For Phase 1, create a zip file "your-wpi-login_project2_phase1.zip"

  1. All the code for the alternating-bit protocol (any .c/.c++ and .h files), including any code that was pre-provided as part of this project

Combine (via zip or tar) the two zip files for the two phases up into a single archive file named "your-wpi-login_project2.zip".

Submit your document electronically via Instruct Assist (http://cerebro.cs.wpi.edu/cs3516) by 11:59pm on the day the assignment is due. Make sure you choose "Project 2" under project drop-down in IA before uploading the zip file.
