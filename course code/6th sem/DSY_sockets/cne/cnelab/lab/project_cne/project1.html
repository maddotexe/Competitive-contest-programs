Description

In this assignment you will be asked to implement an HTTP client and server running a simplified version of the HTTP/1.0 protocol. This project will be completed in either C/C++ using the Unix Socket commands.

Details

  1. The detailed instructions for the project can be downloaded by clicking here
  2. The web client should be able to connect to any URL/port specified as a run-time argument and print (on the screen) the html code that is returned when the call is successful
  3. The web client should print (on the screen) the HTTP code returned by the server (200 for success and 404 for page not found)
  4. The web server should either return the requested file (index.html) with a "200 OK" code or return a "404 page not found" code
  5. This assignment is worth - 100 points
  6. All code must run on the WPI CCC machines

Submission

As a part of the submission you need to provide the following:

  1. All the code for the web client and the web server (.c and .h files)
  2. A makefile that compiles the client and the server code (Tutorial on writing a makefile)
  3. A ReadMe (txt) file that spells out exactly how to compile and run the client and the server
  4. A PDF file that lists the RTT tables for the actual and your web server. Please write exactly which actual web server you connected to. For example www.cnn.com
Combine (via zip or tar) everything up into a single archive file named "your-wpi-login_project1.zip".

Submit your document electronically via Instruct Assist (http://cerebro.cs.wpi.edu/cs3516) by 11:59pm on the day the assignment is due. Make sure you choose "Project 1" under project drop-down in IA before uploading the zip file.

Grading

Grade breakup (%) assuming all documents are present:

  1. Web client = 45%
  2. Web server = 40%
  3. RTT results = 10%
  4. makefile = 5%

The grade will be a ZERO if:

  1. If the code (client or server) does not compile or gives a run-time error
  2. If README with detailed instructions on compiling the running the code is not present
  3. If makefile is erroneous and the README does not provide a way tell us how to compile and run the code
